# Muonnoi Mobile Design System Lock

Date: 2026-05-11
Scope: iOS + Android app design baseline.

## Brand roles

Primary:
- Azure for voice/action.

Trust:
- Gold only for verified, accepted or trusted state.

Surface:
- Neutral ink/dark surface by default.

Warning:
- Amber for attention.

Error:
- Red only for destructive or failed state.

## Type scale

- Display: 32/38
- Screen title: 26/32
- Section title: 20/26
- Body: 16/24
- Small: 14/20
- Caption: 12/16

Rules:
- No text smaller than 12.
- Buttons must not truncate primary action.
- Error messages must be readable without tooltips.

## Spacing scale

- 4

- 12
- 16
- 20
- 24

- 40

## Components

Required:
- app tab bar
- top app bar
- primary button
- secondary button
- icon button
- quest card
- proof card
- receipt card
- status pill
- permission rationale sheet
- offline banner
- empty state
- error state
- loading skeleton

## Motion and haptics

Allowed:
- route transition
- upload progress
- proof accepted confirmation
- subtle haptic on completed action

Forbidden:
- confetti for financial-like rewards
- addictive streak animation
- attention-grabbing loops

## Accessibility

- Dynamic type must be supported.
- Touch target minimum: 44 x 44.
- Color cannot be the only state indicator.
- Permission copy must be plain and direct.
