# 08 — Brand Book Outline: Cấu trúc Brand Book 60–80 trang

> Brand Book là **tài liệu trình bày đẹp** đóng gói các quyết định từ file 02 -> 07. Đối tượng: nhân viên mới, agency outsource, đối tác in ấn, designer freelance, developer.

---

## A. HƯỚNG DẪN

- Brand Book khác file spec (07): file 07 là **kỹ thuật**, brand book là **truyền cảm hứng + áp dụng nhanh**.
- Tối ưu: 60–80 trang. Dưới 40 = thiếu. Trên 100 = không ai đọc.
- Phát hành 2 dạng: **PDF print-ready** (interactive TOC, hyperlinked) + **Notion / Frontify / Confluence** (sống, update được).
- Mỗi phiên bản có **changelog** ở cuối.
- Owner: Designer trưởng. Approver: Founder + Brand Strategist.

---

## B. CẤU TRÚC CHUẨN — 12 chương

### Chương 0 — Cover + Welcome (3 trang)

```
P1: Cover
- Logo lớn + tên "Brand Book"
- Phiên bản + ngày phát hành
- Tagline primary

P2: Welcome letter từ Founder
- 200–300 từ
- Vì sao brand book này quan trọng
- Mong đợi với người đọc

P3: Table of Contents (hyperlinked)
```

### Chương 1 — Brand Foundation (8–10 trang)

```
1.1 Câu chuyện thương hiệu (Story) — 1 trang
    Câu chuyện 200–300 từ về lý do brand ra đời.

1.2 Mission, Vision (1 trang)
    Lấy từ file 05.

1.3 Core Values (3–5 trang)
    Mỗi value 1 trang: định nghĩa, hành vi mong đợi, ví dụ thực tế.

1.4 Positioning (1 trang)
    Câu định vị + diagram cạnh tranh.

1.5 Brand Persona (2 trang)
    Persona card đầy đủ.

1.6 Brand Promise + RTB (1 trang)
```

### Chương 2 — Verbal Identity (8–10 trang)

```
2.1 Voice (4 trục) — 1 trang
2.2 Tone palette — 2 trang
2.3 Naming convention — 2 trang
2.4 Do/Don't word list — 2 trang
2.5 Tagline system — 1 trang
2.6 Mẫu câu chuẩn 8 context — 2 trang

(Lấy từ file 06)
```

### Chương 3 — Logo System (8–10 trang)

```
3.1 Logo introduction — 1 trang
    Câu chuyện đằng sau logo, ý nghĩa biểu tượng.

3.2 Logo variants — 2 trang
    Primary, secondary, stacked, monogram, wordmark, mono.

3.3 Construction & grid — 1 trang

3.4 Clear space & min size — 1 trang

3.5 Color variants — 1 trang

3.6 Background usage — 1 trang

3.7 Co-branding rules — 1 trang

3.8 Don'ts — 1 trang (ví dụ trực quan)
```

### Chương 4 — Color & Typography (6–8 trang)

```
4.1 Primary palette — 1 trang
4.2 Secondary & neutral palette — 1 trang
4.3 Semantic colors (success, warning, error, info) — 1 trang
4.4 Accessibility — 1 trang (contrast ratios)
4.5 Typography scale — 2 trang
4.6 Font pairing & usage — 1 trang
4.7 Code font (monospace) — 1 trang (nếu tech)
```

### Chương 5 — Imagery & Motion (4–6 trang)

```
5.1 Photography style — 2 trang
5.2 Illustration style — 1 trang
5.3 Iconography — 1 trang
5.4 Motion principles — 1 trang
5.5 Asset library access — 1 trang
```

### Chương 6 — Design System Snapshot (4–6 trang) — NEW in v2.0

```
6.1 Design token overview — 1 trang
6.2 Spacing & layout scale — 1 trang
6.3 Component principles — 2 trang
6.4 Dev handoff link — 1 trang (Storybook / Figma Dev Mode URL)
```

### Chương 7 — Product Brand Snapshot (4–6 trang) — NEW in v2.0

```
7.1 Feature naming convention — 1 trang
7.2 Onboarding tone & copy — 2 trang
7.3 Error state examples — 2 trang
7.4 API / SDK branding — 1 trang
```

### Chương 8 — Digital Applications (6–8 trang)

```
8.1 Website — 2 trang

8.3 Email templates — 1 trang
8.4 Presentation template — 1 trang
8.5 App / Product UI — 2 trang
```

### Chương 9 — Print Applications (4–6 trang)


### Chương 11 — Governance & Contacts (2–4 trang)
