## 4. Architecture Decisions (Locked)

| Decision | Value | Rationale |
|----------|-------|-----------|
| Game format | Single-file `index.html` per game | Zero build step, instant deploy, easy copy-paste |
| SDK | `plays-sdk.js` (vanilla JS, no framework) | 11KB, no deps, works in any HTML file |
| Scoring (guest) | `localStorage` (`plays_scores`) | Works offline, zero API calls |
| Scoring (member) | D1 via Workers API | Server-authoritative, anti-cheat |
| Styling | Inline + `plays.css` | Per-game override possible, shared base |
| Deploy target | Cloudflare Pages | CDN edge, free tier, custom domain |

## 7. How to Add a New Game

1. Create `plays/games/{game-id}/index.html` (copy from existing game as template)
2. Add entry to `plays/catalog.js` (incremental `id`, set `live: true`)
3. Run `node --check plays/games/{game-id}/index.html` (extract JS block if needed)
4. Run verification commands from Section 2
5. `git add . && git commit -m "feat(plays): add {game-name}"`
6. `bash deploy.sh`
