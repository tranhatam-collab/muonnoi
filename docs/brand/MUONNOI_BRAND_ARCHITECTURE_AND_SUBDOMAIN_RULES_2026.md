## Immutable global core
These do not change per subdomain:
- Master brand name.
- MN mark.
- Privacy-first posture.
- No tracker-based growth.
- No fake play-to-earn.
- No public claim without evidence.
- Language switch labels.
- Security and trust links.
- Founder/Product final sign-off.

## Flexible module expression
Subdomains may adjust:
- page title and descriptor
- icon set
- hero image or editorial subject
- local task examples
- module-specific CTA
- proof or onboarding steps

Subdomains may not adjust:
- master trust claim
- payment/investment disclaimers
- privacy promises
- legal entity language
- brand name spelling
- visual palette beyond approved accents

## Required links per subdomain homepage
Every subdomain homepage must link to:
- `https://www.muonnoi.org/`
- `https://www.muonnoi.org/ecosystem/`
- `https://www.muonnoi.org/security/`
- `https://www.muonnoi.org/guide/`
- `https://docs.muonnoi.org/` when docs are live.

## Launch gate
Before a subdomain can be called brand-ready:
- Cloudflare project/domain verified.
- Source folder verified.
- Brand name and descriptor checked.
- Language labels checked.
- No unverified claim.
- Link back to master public shell.
- Security/trust link present.
- Team owner named.
