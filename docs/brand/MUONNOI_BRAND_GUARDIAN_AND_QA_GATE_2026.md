# MUONNOI BRAND GUARDIAN AND QA GATE · 2026

## Purpose
This file turns brand into an implementation gate.

No public page, subdomain homepage, mobile screen, docs package or launch report should be called brand-ready until it passes this gate.

## Owners
Founder/Product:
- final meaning, naming, launch claim and public positioning.

Brand Guardian:
- style lock, voice, logo, language, subdomain consistency and quarterly audit.

Web/Public:
- website copy, metadata, sitemap, redirects and visual consistency.

Docs:
- source-of-truth publishing and cross-file consistency.

Mobile:
- app labels, navigation, permission copy, notifications and store listing language.

Security/Legal:
- privacy, investment, payment, proof, legal claims and data wording.

QA/Release:
- route matrix, metadata, language, accessibility and live evidence.

## Mandatory gate
For every public surface, check:
- Official name is correct.
- MN mark usage is correct.
- Language labels are `Tiếng Việt` and `English`.
- One H1 only.
- Title and description exist.
- Canonical exists.
- OG title, description and image exist.
- No unverified public claim.
- No ROI or earning promise.
- No direct CTA to unverified subdomain.
- Mobile text fits and does not overlap.
- Vietnamese diacritics render correctly.
- Footer links include public launch sources or shared script adds them.
- Security/trust route is reachable.

## Report shape
Every brand QA report must include:
- Verdict
- Evidence checked
- Pass
- Fail
- Blocked by Founder
- Blocked by external asset
- True state
- Team command
- Hard stop

## Hard stop
Do not call a page brand-ready if:
- the brand name is wrong
- visible language labels are abbreviated
- unverified partner/user/host/payment claim appears
- investment copy implies return or guaranteed upside
- route is missing
- metadata is missing
- mobile layout clips Vietnamese text
- logo or mark is inconsistent with this lock

## Quarterly audit
Run every quarter:
- public homepage audit
- top subdomain homepage audit
- docs source-of-truth audit
- mobile store listing audit
- footer/header link audit
- claim safety audit
