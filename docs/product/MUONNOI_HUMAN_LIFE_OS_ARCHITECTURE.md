# MUONNOI HUMAN LIFE OS ARCHITECTURE

## Layers
1. Public understanding: `muonnoi.org`
2. Social operation: `app.muonnoi.org`
3. AI/workflow support: `ai.muonnoi.org`
4. Documentation and policies: `docs.muonnoi.org`
5. Trust and verification: `verify.muonnoi.org`, `trust.muonnoi.org`
6. Life Quest OS: quests under root and future subdomains
7. Work, family, living, investment and project layers

## Architecture rule
Each module may have its own subdomain, but all modules must share one trust policy, one public language standard, one claim-safety standard and one release evidence standard.
