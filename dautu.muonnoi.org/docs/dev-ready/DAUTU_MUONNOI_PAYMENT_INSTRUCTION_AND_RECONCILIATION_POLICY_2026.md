> **Policy for Accounting + Dev + Legal.** Không nhận tiền nếu vi phạm bất kỳ điều kiện nào.

## 1. PAYMENT INSTRUCTION GATE

### 1.1. Điều kiện phát hành instruction

**BẮT BUỘC tất cả:**
1. Investor status = `approved`
2. Project investment_status = `round_open` hoặc `interest`
3. Investment access status = `approved`
4. Legal review status = `approved`
5. Accounting review status = `approved`
6. NDA signed (if required)
7. Amount approved by admin (dual control)
8. Reference code generated (unique, 16 chars)
9. Expiry date set (max 30 days)
10. Source account from `pay.iai.one` verified record

### 1.2. Reference Code Format

```
INV-{PROJECT_SLUG}-{YYYYMM}-{SEQUENCE}
Example: INV-DSTS-202606-0001
```

### 1.3. Instruction Content

```
Pháp nhân nhận tiền: [Name] — [Entity ID]
Mục đích thanh toán: Đầu tư / Tài trợ dự án [Project Name]
Mã tham chiếu: [Reference Code]
Số tiền được phê duyệt: [Amount] [Currency]
Đơn vị tiền: VND / USD
Thời hạn thanh toán: [Expiry Date]

⚠️ CẢNH BÁO:
- Chỉ chuyển đúng số tiền được phê duyệt
- Ghi rõ mã tham chiếu trong nội dung chuyển khoản
- Không chuyển vào tài khoản khác ngoài tài khoản được chỉ định
- Thông tin tài khoản lấy từ pay.iai.one, không từ email/Zalo
- Nếu nghi ngờ, liên hệ admin qua kênh chính thức
```

## 2. RECONCILIATION WORKFLOW

### 2.1. Step-by-Step

```
Step 1: Bank Statement Received
  → Accounting operator nhập vào hệ thống
  → Fields: date, amount, currency, sender name, reference code, bank

Step 2: Auto-Match
  → System match reference_code với payment_instructions
  → Nếu match: tạo draft transaction
  → Nếu không match: flag manual review

Step 3: Amount Verification
  → So sánh amount với amount_approved
  → Nếu đúng: proceed
  → Nếu sai: lock, yêu cầu investor xác nhận

Step 4: Dual Control Review
  → Accounting operator review
  → Accounting manager approve
  → Nếu reject: ghi lý do, thông báo investor

Step 5: Ledger Recording
  → Tạo transaction status = 'confirmed'
  → Tạo 2+ ledger_entries (double-entry)
  → Update wallet buckets
  → Create audit log

Step 6: Investor Notification
  → Email qua mail.iai.one
  → Update member dashboard real-time
```

### 2.2. Mismatch Handling

| Case | Action | Status |
|------|--------|--------|
| Sai số tiền | Lock, yêu cầu xác nhận | `locked` |
| Sai reference | Manual match by admin | `pending_reconciliation` |
| Sai người gửi | AML check, lock | `locked` |
| Duplicate | Replay guard, reject | `rejected` |
| Quá hạn | Reject, yêu cầu re-issue instruction | `expired` |

## 3. REFUND POLICY

### 3.1. Điều kiện refund

1. Legal approval
2. Accounting approval
3. Entity approval (pháp nhân nhận tiền)
4. Investor request có lý do hợp lý
5. Chưa có capital deployed (hoặc số chưa deploy)

### 3.2. Refund process

```
Investor request refund
  → Admin review
    → Legal check
      → Accounting confirm số dư
        → Entity approve
          → Tạo reversal transaction
            → Ghi ledger (reversal entry)
              → Process bank transfer
                → Confirm completion
                  → Audit log
```

---

## 4. DOCUMENT REQUIREMENTS

### 4.1. Mỗi confirmed transaction phải có

```
□ Bank statement (uploaded to R2)
□ Payment instruction (generated by system)
□ Investor acknowledgement (if applicable)
□ Admin approval log (audit_logs)
□ Ledger entries (balanced)
```

### 4.2. Retention

- Bank statements: 10 năm
- Ledger entries: vĩnh viễn (append-only)
- Audit logs: 10 năm
- Investor KYC: 5 năm sau khi quan hệ kết thúc
