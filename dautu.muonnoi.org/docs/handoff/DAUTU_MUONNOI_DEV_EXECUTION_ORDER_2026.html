⚡ DEV EXECUTION ORDER

Thứ tự thực thi bắt buộc cho team dev dautu.muonnoi.org — đảm bảo đúng doctrine, đúng process, và đúng quality

1. EXECUTION PRINCIPLES

1.1. Core principles

  • Doctrine-first — Doctrine là source of truth
  • Sequential execution — Không skip phases
  • Quality gates — Mỗi phase phải pass QA
  • No partial deployment — Không live từng phần rác
  • Documentation first — Docs trước khi code

2. PHASE EXECUTION ORDER

2.1. Phase 1 — Lock Doctrine

Objective: Khóa định nghĩa đầu tư, tone, và CTA registry

Tasks:

  1. Review và approve Master Doctrine document
  2. Khóa định nghĩa đầu tư
  3. Khóa anti-definition
  4. Khóa tone guidelines
  5. Khóa CTA registry
  6. Khóa forbidden words list

Deliverable: Doctrine approved and locked

Gate: Founder sign-off required

2.2. Phase 2 — Lock Content Source

Objective: Tạo hệ thống content source chuẩn

Tasks:

  1. Tạo /content/vi.json
  2. Tạo /content/en.json
  3. Tạo /content/seo.vi.json
  4. Tạo /content/seo.en.json
  5. Map toàn bộ text hiện tại
  6. Remove hard-coded text

Deliverable: Content source system complete

Gate: Content lead sign-off required

2.3. Phase 3 — Rebuild Homepage

Objective: Rebuild homepage theo doctrine

Tasks:

  1. Sửa H1 theo doctrine
  2. Sửa hero theo doctrine
  3. Sửa CTA theo doctrine
  4. Thêm section doctrine
  5. Thêm section risk
  6. Thêm journey map
  7. Thêm final CTA đúng

Deliverable: Homepage rebuilt per doctrine

Gate: Founder + Content lead sign-off required

2.4. Phase 4 — Rebuild 7 Routes

Objective: Rebuild 7 public routes còn lại

Tasks:

  1. Rebuild /du-an/
  2. Rebuild /hinh-thuc-dong-hanh/
  3. Rebuild /quy-trinh-review/
  4. Rebuild /governance/
  5. Rebuild /faq/
  6. Rebuild /dang-ky-quan-tam/
  7. Rebuild /legal/
  8. Đồng bộ nav/footer
  9. Đồng bộ metadata
  10. Đồng bộ bilingual content

Deliverable: 7 routes rebuilt per doctrine

Gate: Content lead sign-off required

2.5. Phase 5 — QA 4 Pass

Objective: QA qua 4 layers

Tasks:

  1. AI pass — Automated content check
  2. Editor/Content pass — Manual content review
  3. SEO pass — SEO validation
  4. QA/Dev pass — Technical QA

Deliverable: All 4 QA passes complete

Gate: All 4 QA leads sign-off required

2.6. Phase 6 — Live Only When Clean

Objective: Deploy khi tất cả clean

Tasks:

  1. Final deployment preparation
  2. Staging deployment
  3. Final staging QA
  4. Production deployment
  5. Post-deployment verification

Deliverable: Clean production deployment

Gate: Founder final approval required

5. FORBIDDEN EXECUTION PATTERNS

5.1. Không được skip phases

KHÔNG ĐƯỢC:

  • Skip Phase 1 (Doctrine lock)
  • Skip Phase 2 (Content source)
  • Skip Phase 5 (QA 4 pass)
  • Deploy từng phần rác
  • Live khi chưa QA xong

5.2. Không được violate doctrine

KHÔNG ĐƯỢC:

  • Implement content vi phạm doctrine
  • Use CTA vi phạm CTA registry
  • Use từ vi phạm forbidden words
  • Display payment info public
  • Make claims vi phạm legal posture
