🏗️ INFORMATION ARCHITECTURE

Cấu trúc thông tin bắt buộc cho dautu.muonnoi.org — đảm bảo user journey rõ ràng, doctrine-first, và SEO-friendly

1. SITE STRUCTURE OVERVIEW

1.1. Primary navigation structure

  • Homepage (/) — Entry point, doctrine-first
  • Projects (/du-an/) — Project catalog (not sales catalog)
  • Participation (/hinh-thuc-dong-hanh/) — Participation levels
  • Process (/quy-trinh-review/) — Review process
  • Governance (/governance/) — Governance & legal posture
  • FAQ (/faq/) — Common questions
  • Contact (/dang-ky-quan-tam/) — Interest form (not payment)
  • Legal (/legal/) — Legal disclaimers

2. USER JOURNEY ARCHITECTURE

2.1. Primary user flow

Awareness (Homepage)
    ↓
Understanding (Philosophy Sections)
    ↓
Interest (Projects/Participation)
    ↓
Engagement (Interest Form)
    ↓
Review (Private Review Process)
    ↓
Participation (If Suitable)

2.2. Secondary user flows

Flow A: Quick FAQ Access

Homepage → FAQ → Specific Answer → Back to Interest

Flow B: Governance Check

Homepage → Governance → Legal → Interest Form

Flow C: Project Deep Dive

Homepage → Projects → Specific Project → Participation → Interest Form

3.2. Level 2: Supporting Pages

4. CONTENT HIERARCHY

4.1. Homepage content hierarchy

  1. Hero Section — Primary doctrine statement
  2. Philosophy Section — Why learn first
  3. Anti-Definition Section — What it's NOT
  4. Journey Section — User journey map
  5. Value Section — True value proposition
  6. Risk Section — Honest risk disclosure
  7. System Section — Projects/participation overview
  8. CTA Section — Getting started

4.2. Supporting page content hierarchy

Projects Page:

  1. Page title & intro
  2. Project categories/lanes
  3. Project cards (overview)
  4. Participation levels
  5. Next steps CTA

Participation Page:

  1. Page title & intro
  2. Participation philosophy
  3. Level descriptions (Learn → Review → Participate)
  4. Suitability criteria
  5. Next steps CTA

5. NAVIGATION ARCHITECTURE

5.1. Main navigation

  • Logo — Links to homepage
  • Nav Items — 6-7 items max
  • Language Switcher — VI/EN toggle
  • CTA Button — "Bày tỏ quan tâm" / "Express Interest"

5.2. Footer navigation

  • Primary Links — Main pages
  • Legal Links — Legal, Privacy, Terms
  • Contact Info — Minimal contact details
  • Copyright — Standard copyright

5.3. Breadcrumb navigation

Use breadcrumbs for:

  • Project detail pages (if implemented)
  • FAQ category pages (if implemented)
  • Multi-level content pages

6. INTERNAL LINKING STRATEGY

6.1. Contextual linking

  • Link from philosophy to participation
  • Link from risk to legal
  • Link from projects to participation
  • Link from FAQ to relevant sections

6.2. CTA linking

  • Primary CTA → Interest form
  • Secondary CTA → Relevant information page
  • Tertiary CTA → FAQ or Governance

7. CONTENT MODEL ARCHITECTURE

7.1. Page template structure

Page Template:
├── Header (Navigation)
├── Hero Section (Page-specific)
├── Main Content
│   ├── Introduction
│   ├── Core Sections (3-6 sections)
│   ├── Supporting Content
│   └── CTA Section
├── FAQ Section (if applicable)
├── Legal Note (if applicable)
└── Footer

7.2. Section template structure

Section Template:
├── Section Header (H2)
├── Section Intro (1-2 paragraphs)
├── Section Content
│   ├── Sub-sections (H3)
│   ├── Lists/Tables
│   └── Visual Elements
└── Section CTA (if applicable)

9. URL ARCHITECTURE

9.1. URL structure rules

  • Lowercase — All URLs lowercase
  • Hyphens — Use hyphens, not underscores
  • Trailing slash — Include trailing slash
  • Short & descriptive — Keep URLs concise
  • No special characters — Avoid special chars

9.2. URL examples

✅ Correct:
/du-an/
/hinh-thuc-dong-hanh/
/quy-trinh-review/
/governance/
/faq/
/dang-ky-quan-tam/
/legal/

❌ Incorrect:
/Du-An/
/hinh_thuc_dong_hanh/
/QuyTrinhReview/
/governance?page=1
/faq?id=123

10. MOBILE INFORMATION ARCHITECTURE

10.1. Mobile navigation

  • Hamburger menu — Collapsible navigation
  • Sticky CTA — Primary CTA always visible
  • Back navigation — Clear back buttons
  • Touch targets — Min 44x44px

10.2. Mobile content hierarchy

  • Progressive disclosure — Show key info first
  • Collapsible sections — Expandable content
  • Sticky headers — Section headers visible
  • Optimized reading — Readable font sizes

11. SCHEMA.ORG ARCHITECTURE

11.1. Required schema types

  • Organization — Site-wide
  • WebSite — Site-wide
  • FAQPage — FAQ page
  • BreadcrumbList — Breadcrumb pages

11.2. Optional schema types

  • Article — Blog/content pages
  • Event — Event pages (if applicable)
  • Person — Team pages (if applicable)

12. ACCESSIBILITY ARCHITECTURE

12.1. Semantic structure

  • Header — <header> element
  • Navigation — <nav> element

  • Footer — <footer> element

12.2. ARIA architecture

  • Landmarks — ARIA landmarks
  • Labels — ARIA labels for interactive elements
  • Descriptions — ARIA descriptions where needed
  • Live regions — For dynamic content
